# llm-wiki-ingest

인자: ingest할 `raw/` 안의 파일 경로(들). 인자가 없으면 `raw/`에서
`wiki/sources/`에 대응 페이지가 없는 파일을 찾아 사용자에게 확인받는다.

전체 규칙은 저장소 루트의 `AGENTS.md`를 따른다. 절차:

## 1. 소스 읽기

- 대상 파일 전체를 읽는다. 마크다운에 이미지가 포함되어 있으면 텍스트를 먼저
  읽고, 이해에 필요한 이미지를 `raw/assets/`에서 개별적으로 열어 본다.
- 소스의 종류(매뉴얼 페이지, 논문, 예제 스크립트, 포럼 글 등), 날짜, 저자 등
  메타데이터를 파악한다.

## 2. 핵심 확인

- 핵심 내용 요지(3~7개)와, 이 소스가 건드릴 위키 페이지 목록
  (새로 만들 페이지 / 갱신할 페이지)을 사용자에게 간결히 보고한다.
- 사용자가 강조점을 조정하면 반영한다. 사용자가 "그냥 진행해"라고 하면
  이 단계는 이후 ingest에서 생략해도 된다.

## 3. 위키 반영 (순서대로)

1. **`wiki/sources/<이름>.md`** — 소스 요약 페이지 작성. 포함할 것:
   원본 경로(`raw/...`), 소스 종류·출처, 핵심 내용 요약, 이 소스가 기여한
   위키 페이지 링크 목록.
2. **`wiki/reference/`, `wiki/concepts/`** — 소스가 다루는 개체·개념 페이지를
   생성 또는 갱신한다. 기존 페이지와 상충하는 내용은 `> ⚠️ 상충:` 블록으로
   양쪽을 기록. 새 페이지는 반드시 관련 기존 페이지들과 상호 링크한다.
3. **`wiki/overview.md`** — 이 소스로 전체 그림이 바뀌면 갱신한다.
4. **`wiki/index.md`** — 새 페이지 추가, 갱신된 페이지의 한 줄 요약 조정,
   카테고리별 개수와 "마지막 갱신" 날짜 갱신.
5. **`wiki/log.md`** — `## [YYYY-MM-DD] ingest | <소스 제목>` 엔트리를 맨 아래에
   추가하고, 만들거나 고친 페이지를 목록으로 적는다.

## 4. 보고

만든 페이지 / 고친 페이지를 링크와 함께 사용자에게 요약 보고한다.
커밋은 사용자가 요청할 때만 한다.
