# llm-wiki-lint

인자(선택): 점검 범위 (예: `reference/`만). 없으면 위키 전체.
전체 규칙은 저장소 루트의 `AGENTS.md`를 따른다. 절차:

## 1. 점검

`wiki/index.md`와 대상 페이지들을 읽고 다음을 확인한다:

- **모순**: 페이지 간 서로 충돌하는 주장 (표시 안 된 것)
- **낡은 주장**: 나중에 ingest된 소스가 갈아엎은 내용이 남아 있는 페이지
- **고아 페이지**: index 외에 어떤 페이지도 링크하지 않는 페이지
- **깨진 링크**: 존재하지 않는 파일을 가리키는 링크
  (`grep -roh "](\.\./[^)]*\.md\|]([^):/]*\.md" wiki/` 결과를 실제 파일과 대조)
- **index 불일치**: index에 없는 페이지, index에만 있고 실제로는 없는 페이지,
  개수·요약·날짜가 실제와 다른 항목
- **누락 페이지**: 여러 페이지에서 반복 언급되지만 자체 페이지가 없는 개체·개념
- **데이터 공백**: 위키가 답하지 못하는 자연스러운 질문들 — ingest할 소스나
  웹 검색으로 채울 만한 것을 제안

## 2. 보고

발견 사항을 심각도 순으로 정리해 보고한다. 각 항목에 제안 조치를 붙인다.
자동으로 고치지 말고 사용자 승인을 받는다 (기계적으로 명백한 것 —
index 개수 불일치, 깨진 링크 등 — 은 묶어서 일괄 승인 요청).

## 3. 수정 및 기록

승인된 항목을 수정하고, `wiki/log.md`에
`## [YYYY-MM-DD] lint | <범위>` 엔트리로 점검 결과와 수정 내역을 기록한다.
