# llm-wiki-query

인자: 질문. 전체 규칙은 저장소 루트의 `AGENTS.md`를 따른다. 절차:

## 1. 탐색

- `wiki/index.md`를 먼저 읽어 관련 페이지를 고른 뒤 그 페이지들을 읽는다.
- 필요하면 `wiki/` 전체에 grep으로 키워드 검색을 보충한다.
- 페이지가 인용한 `raw/` 원본까지 내려가 확인해야 정확한 답이 되면 그렇게 한다.

## 2. 답변

- 위키/소스에 근거해 답하고, 근거 페이지를 링크로 인용한다.
- 위키에 근거가 없는 부분은 LLM 자체 지식임을 명시해 구분한다.
- 위키가 답을 못 주는 질문이면 그렇다고 말하고, 어떤 소스를 ingest하면
  답할 수 있게 되는지 제안한다.
- 질문 성격에 따라 비교표, 코드 예제 등 적절한 형태를 쓴다.

## 3. 파일링 (선택)

답변이 보존 가치가 있으면(비교·분석·여러 페이지의 종합 등) `wiki/guides/`에
페이지로 저장할지 사용자에게 제안한다. 저장 시:

1. `wiki/guides/<이름>.md` 작성 — 답변 내용 + 근거 페이지 링크.
2. 근거가 된 페이지들에서 이 guide로의 링크 추가 (고아 페이지 방지).
3. `wiki/index.md` Guides 섹션에 추가.
4. `wiki/log.md`에 `## [YYYY-MM-DD] query | <질문 요지>` 엔트리 추가.
